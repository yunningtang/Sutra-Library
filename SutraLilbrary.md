# 佛经念诵 App — PRD 

## 一、产品概述

### 1.1 定位

一款极简的**佛经念诵工具**。核心功能三件事：**读经、计数、查义**。不社交，不游戏化，不花哨。

首发经文为《大乘离文字普光明藏经》，架构支持后续扩充任意经文（《地藏菩萨本愿功德经》《金刚般若波罗蜜经》《心经》《药师经》等）。

### 1.2 设计原则

- **极简克制**：无 icon、无 emoji、无插图、无装饰性元素。所有信息用文字 + 留白传达。
- **经文为主角**：整个 App 是为阅读经文服务的，UI 退到最后面。
- **不游戏化**：没有连续打卡、没有成就徽章、没有问候语、没有每日一句。只有朴素的计数。
- **可扩展**：架构层面支持多部经文，新增一部经 = 新增一份 JSON 数据。

### 1.3 经文库规划

| 阶段 | 经文 | 字数 | 说明 |
|------|------|------|------|
| v1.0 | 《大乘离文字普光明藏经》 | ~2200 | 短经，首发验证 |
| v1.1 | 《般若波罗蜜多心经》 | ~260 | 最短经，用户量大 |
| v1.2 | 《金刚般若波罗蜜经》 | ~5000 | 中篇，需分品导航 |
| v2.0 | 《地藏菩萨本愿功德经》 | ~12000 | 长篇，需分卷/分品 |
| v2.x | 《药师琉璃光如来本愿功德经》 | ~5000 | |
| v2.x | 《佛说阿弥陀经》 | ~1800 | |
| ... | 持续扩充 | | 架构不变，只加数据 |

## 二、信息架构

```
App
├── 经库（首页）
│   └── 经文列表（卡片式，点击进入某部经）
│
├── 经文详情页（进入某部经后的主页面）
│   ├── 念诵入口
│   ├── 累计诵读次数（朴素数字）
│   ├── 经文全文浏览入口
│   └── 词汇注释入口
│
├── 念诵页（全屏沉浸）
│   ├── 经文 + 拼音
│   ├── 自动/手动滚动
│   ├── 长按注释
│   ├── 字号调节
│   └── 念诵完成 → 计数 +1
│
├── 计数器（念珠模式，独立工具）
│
├── 我的
│   ├── 登录/注册
│   ├── 诵读记录（按经文分组的历史）
│   ├── 设置（字号、拼音、滚动速度、深色模式）
│   └── 关于
│
└── 底部导航（三个文字 Tab）
    经库 · 计数 · 我的
```

**关键决策**：
- 底部只有 3 个 Tab，用纯文字，不用任何图标
- 没有"首页"概念，进 App 直接看到经库
- 没有打卡日历、没有连续天数、没有里程碑

### 3.3 念诵页（核心）

全屏沉浸式，无底部导航，无顶部标题栏干扰。

```
┌──────────────────────────────┐
│                              │
│  关闭                    拼  │  ← 顶部工具行：纯文字
│                          Aa  │     关闭 / 拼音开关 / 字号
│                              │
│                              │
│  rú shì wǒ wén              │  ← 拼音（可关闭）
│  如 是 我 闻 ：              │  ← 经文
│                              │
│  yī shí，fó zài wáng shè    │
│  一 时 ， 佛 在 王 舍        │
│                              │
│  chéng qí shé jué shān      │
│  城 耆 阇 崛 山 中 ，        │
│                              │
│  yǔ dà pú sà wú liàng       │
│  与 大 菩 萨 无 量            │
│                              │
│  bǎi qiān yì nà yóu tā     │
│  百 千 亿 那 由 他            │
│                              │
│  shù                         │
│  数 ， 皆 是 大 智            │
│                              │
│  ......                      │
│                              │
│                              │
│  ━━━━━━━━━░░░░░░░  38%      │  ← 细线进度条
│                      完成诵读 │  ← 到底后出现
└──────────────────────────────┘
```

**经文排版**：
- 背景色：`#f7f5f0`（微暖米色，区别于其他页面，营造阅读氛围）
- 每行 7-10 个汉字（屏幕宽度自适应）
- 汉字默认 22px，用户可调 18 / 20 / 22 / 24 / 26
- 汉字字间距 4-6px
- 拼音 11px，色值 `#999`，在汉字正上方逐字对齐
- 行高：无拼音时 2.0，有拼音时 2.8
- 字体可选：黑体（默认）或 宋体（设置中切换）
- 正文区域水平内边距 24px（比其他页面略大，为阅读留气）
- 段落之间留一个空行

**顶部工具行**：
- 半透明背景，向下滚动后逐渐隐藏，点击屏幕顶部重新出现
- "关闭"：纯文字，点击返回详情页（带确认，如果念诵中途退出）
- "拼"：切换拼音显隐
- "Aa"：点击弹出字号调节浮层（5 档可选，纯文字列表）

**自动滚动**：
- 向上轻滑触发自动滚动模式，再次触摸暂停
- 或在设置中默认开启
- 速度三档：缓 / 中 / 快

**长按注释**：
- 长按任意汉字 300ms 触发
- 弹出浮层，出现在手指上方，白底，12px 圆角，轻微阴影
- 浮层内容：

```
┌────────────────────────┐
│  耆阇崛  qí shé jué    │  ← 词 + 拼音
│                        │
│  灵鹫山。梵文           │  ← 简短释义
│  Gṛdhrakūṭa 音译。     │
│  古印度王舍城说法处。   │
└────────────────────────┘
```

- 入场：opacity 0→1 + translateY(8px)→0，300ms spring
- 松手或点击外部关闭
- 如果该字不在词汇表中，则只显示拼音
- 浮层宽度自适应内容，最大不超过屏幕宽度 - 40px

**念诵完成**：
- 滚动到底部后，出现"完成诵读"文字按钮
- 点击后：
  1. 页面渐变为白底
  2. 居中显示"诵读圆满"四个字（24px，主色 Green）
  3. 下方"第 109 部"（计数 +1 后的数字）
  4. 再下方"回向"文字链接（点击展开回向偈文本）
  5. "返回"文字链接
- 整个完成页没有任何动画华丽效果，就是安静的文字

### 3.4 计数器（念珠模式）

独立的计数工具，不绑定任何经文。可用于持咒、念佛号等。

```
┌──────────────────────────┐
│                          │
│                          │
│                          │
│          32              │  ← 当前计数，48px/700
│         / 108            │  ← 目标值，20px，灰色
│                          │
│                          │
│     ┌──────────┐         │
│     │          │         │  ← 大圆形点击区域
│     │          │         │     屏幕宽度 55%
│     │          │         │     绿色细描边(2px)
│     │          │         │     点击区域 +1
│     │          │         │
│     └──────────┘         │
│                          │
│                          │
│  重置              目标   │  ← 底部两个文字链接
│                          │
│                          │
│  经库        计数     我的 │
└──────────────────────────┘
```

**交互**：
- 点击大圆区域 +1
- 点击反馈：圆圈 border 短暂加粗（2px → 3px，100ms）+ 数字微弹
- 有触觉震动（轻）
- 满 108（或自定义目标）时：数字短暂变为主色 Green + 较强震动
- "重置"需二次确认（弹出底部确认条："确定重置？ 确定 / 取消"）
- "目标"点击弹出底部弹窗，输入框设定目标数
- 圆形按钮周围没有进度环（极简原则），只靠数字传达

信息层级完全依靠：**字号大小、字重粗细、颜色深浅、留白间距**。

## 五、动画规范

### 5.1 原则

动画是为了让操作感觉自然流畅，不是为了好看。**宁可没有也不过度。**

### 5.2 缓动

| 名称 | 曲线 | 用途 |
|------|------|------|
| 标准 | `cubic-bezier(0.23, 1, 0.32, 1)` | 页面转场、浮层 |
| 弹性 | `cubic-bezier(0.34, 1.56, 0.64, 1)` | 数字跳变 |

### 5.3 转场

| 场景 | 动画 | 时长 |
|------|------|------|
| Tab 切换 | crossfade | 200ms |
| 进入念诵页 | 从底部滑入 | 350ms |
| 退出念诵页 | 向底部滑出 | 300ms |
| 注释浮层出现 | opacity + translateY(8px) | 250ms |
| 注释浮层消失 | opacity | 150ms |
| 底部确认条 | 从底部滑入 | 250ms |

### 5.4 反馈

| 场景 | 动画 |
|------|------|
| 按钮 press | scale(0.95)，松开恢复 |
| 计数 +1 | 数字 scale(1.08) → 1.0，200ms |
| 满目标数 | 数字变绿 500ms，然后恢复 |
| 卡片 press | translateY(-1px)，松开恢复 |

**不使用的动画**：粒子效果、光芒扩散、弹幕、confetti、翻牌、3D 变换。

## 六、经文数据架构

### 6.1 经文配置（多经文扩展核心）

每部经是一个独立 JSON 文件，App 只需加载对应文件即可。

```
/data/sutras/
├── manifest.json          ← 经文目录
├── pglm.json              ← 大乘离文字普光明藏经
├── heart.json             ← 心经
├── diamond.json           ← 金刚经
├── dizang.json            ← 地藏经
└── ...
```

### 6.3 关键拼音注意事项

佛经中有一批字词的读音与现代普通话不同，必须特殊标注：

| 词 | 错误读音 | 正确读音 | 说明 |
|------|------|------|------|
| 南无 | nán wú | ná mó | 梵文 namas 音译 |
| 般若 | bān ruò | bō rě | 梵文 prajñā 音译 |
| 叶 (人名中) | yè | shè | 如"迦叶" kā shè |
| 他 (那由他) | tā | tuō | 梵文音译 |
| 华 (妙华等) | huá | huā | 佛经中多读 huā |

这些需在数据中逐字标注正确拼音，不可依赖自动注音工具。

## 八、技术建议

### 8.1 前端

- React Native（跨平台）或 React H5（先验证）
- 状态管理：Zustand
- 本地存储：AsyncStorage / localStorage
- 经文数据：随 App 本地打包（单部经 JSON < 100KB）

### 8.2 后端（可选 BaaS）

- Supabase 或 Firebase
- 认证：手机号验证码
- 数据库：诵读记录 + 用户信息
- 登录后自动上传本地数据

### 8.3 离线优先

- 经文 100% 本地
- 念诵、计数完全离线
- 登录和同步在有网时自动执行

## 十、约束

- **无任何 icon**，信息层级用字号 + 字重 + 色深 + 留白表达
- **无 emoji**
- **无连续打卡**、无成就徽章、无问候语、无每日一句
- **无游戏化措辞**，如果需要用语，用"诵读圆满""随喜"等朴素佛教用语
- **经文准确性第一**，以《乾隆大藏经》版本为准
- **拼音须人工校对**，尤其是佛经专用读音
- **移动端优先**，触控区 ≥ 44px
- **动画克制**，仅用于过渡和操作反馈，不做装饰性动画
- **多经文架构**，加一部经 = 加一个 JSON，不改代码
